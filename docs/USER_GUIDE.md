## 1. ما هو هذا البرنامج؟

**DICOM Print Server** يعمل كـ **وسيط ذكي** بين جهازك الطبي (أشعة، سونار، CT، MRI) وطابعتك.

```
┌────────────────┐     DICOM      ┌──────────────────────┐     Windows     ┌──────────────┐
│  جهاز طبي     │ ─────────────► │  DICOM Print Server  │ ──────────────► │   طابعة      │
│ (CR/DR/CT/US)  │   بروتوكول    │  (هذا البرنامج)      │   GDI Printing  │  (أي طابعة) │
└────────────────┘               └──────────────────────┘                 └──────────────┘
                                          │
                          ┌───────────────┼────────────────┐
                          ▼               ▼                ▼
                      حفظ JPG        توليد PDF       إشعار WhatsApp
```

**ما الذي يحله هذا البرنامج؟**

معظم الأجهزة الطبية لا تستطيع الطباعة مباشرة على طابعات Windows عادية — فهي تتحدث بروتوكول DICOM فقط. هذا البرنامج يستقبل أوامر DICOM ويُحوّلها إلى طباعة عادية على أي طابعة مثبتة في Windows.

---

## 2. هل يحتاج سيرفر؟

**لا. ليس بالمعنى التقليدي.**

البرنامج يُثبَّت على **نفس الكمبيوتر** المتصل بالطابعة، ويعمل كـ **Windows Service** في الخلفية. لا تحتاج:
- ❌ سيرفر مخصص
- ❌ اشتراك سحابة
- ❌ إنترنت (إلا لإشعارات WhatsApp)
- ❌ قاعدة بيانات منفصلة (SQLite مُضمَّنة)

**ما تحتاجه فعلاً:**

```
┌─────────────────────────────────────────────────────┐
│  كمبيوتر Windows 10/11 عادي                         │
│  (ولو مواصفات متوسطة: 4GB RAM، i3)                  │
│                                                      │
│  ┌────────────────────────────────┐                  │
│  │   DICOM Print Server           │ ← هذا البرنامج  │
│  │   (يعمل كـ Windows Service)    │                  │
│  └────────────────────────────────┘                  │
│                                                      │
│  متصل بالشبكة المحلية للمستشفى/العيادة              │
│  ومتصل بالطابعة (USB أو شبكة)                       │
└─────────────────────────────────────────────────────┘
```

الأجهزة الطبية في نفس الشبكة تُرسل أوامر الطباعة مباشرة إلى هذا الكمبيوتر.

## 3. كيف يعمل بشكل عام؟

### الإعداد الأولي (مرة واحدة)

```
1. تُثبّت البرنامج على كمبيوتر متصل بالطابعة
2. تُضيف في appsettings.json:
   - اسم AET (مثل PRINTER_A)
   - رقم منفذ DICOM (مثل 104 أو 8000)
   - اسم الطابعة في Windows
3. في الجهاز الطبي تضبط:
   - IP: عنوان كمبيوتر البرنامج
   - AET: نفس الاسم في appsettings.json
   - Port: نفس المنفذ
4. تشغّل البرنامج كـ Service
```

### عند كل طباعة (تلقائياً بدون تدخل)

```
1. الجهاز الطبي يُرسل صورة DICOM
2. البرنامج يستقبلها ويُحوّلها
3. يطبعها على الطابعة Windows
4. يحفظ JPG/PDF (إذا كان مُفعَّلاً)
5. يُرسل إشعار WhatsApp (إذا كان مُفعَّلاً)
6. يُسجّل العملية في قاعدة البيانات
```

## 4. التثبيت والتشغيل

### الطريقة 1: تشغيل مباشر (للاختبار)

```cmd
DicomPrintServer.exe
```

ستظهر السجلات مباشرة في نافذة CMD.

### الطريقة 2: تثبيت كـ Windows Service (للإنتاج)

```powershell
# افتح PowerShell كمسؤول

# تثبيت الخدمة
sc create "DicomPrintServer" ^
   binPath= "C:\DicomPrintServer\DicomPrintServer.exe" ^
   DisplayName= "DICOM Print Server" ^
   start= auto

# تشغيل الخدمة
sc start DicomPrintServer

# التحقق من الحالة
sc query DicomPrintServer
```

### إدارة الخدمة

```cmd
sc start DicomPrintServer    # تشغيل
sc stop DicomPrintServer     # إيقاف
sc delete DicomPrintServer   # حذف الخدمة
```

### فتح المنافذ في جدار الحماية (Windows Firewall)

```powershell
# السماح بمنفذ DICOM
netsh advfirewall firewall add rule name="DICOM Print" dir=in action=allow protocol=TCP localport=8000

# السماح بلوحة التحكم
netsh advfirewall firewall add rule name="DICOM Admin" dir=in action=allow protocol=TCP localport=9000
```

## 5. ملف الإعدادات

**الملف:** `appsettings.json` بجانب `DicomPrintServer.exe`

```json
{
  "PrintServer": {

    "Listeners": {
      "PRINTER_A": {
        "AET": "PRINTER_A",
        "Port": 8000,
        "WindowsPrinterName": "HP LaserJet P2035",
        "SaveJpg": true,
        "SavePdf": false,
        "OutputFolder": "C:\\PrintOutput\\PRINTER_A",
        "Gamma": 1.0,
        "Contrast": 1.0,
        "Brightness": 0
      },
      "PRINTER_B": {
        "AET": "PRINTER_B",
        "Port": 8001,
        "WindowsPrinterName": "Microsoft Print to PDF",
        "SaveJpg": false,
        "SavePdf": true,
        "OutputFolder": "C:\\PrintOutput\\PRINTER_B"
      }
    },

    "AdminApi": {
      "Enabled": true,
      "Port": 9000,
      "AdminUsername": "admin",
      "AdminPasswordHash": ""
    },

    "WhatsApp": {
      "Enabled": false,
      "ApiKey": "YOUR_CALLMEBOT_APIKEY",
      "DefaultRecipientPhone": "+966500000000",
      "SendImage": false
    },

    "Trial": {
      "MaxOps": 50,
      "MaxDays": 14,
      "Enabled": true
    }

  }
}
```

### معنى كل حقل

| الحقل | الوصف | مثال |
|-------|-------|------|
| `AET` | اسم الطابعة كما يراها الجهاز الطبي | `PRINTER_A` |
| `Port` | منفذ DICOM (يُفضَّل 8000+ لتجنب صلاحيات) | `8000` |
| `WindowsPrinterName` | اسم الطابعة **بالضبط** من Windows | `HP LaserJet P2035` |
| `SaveJpg` | هل تحفظ صورة JPG لكل صفحة؟ | `true` / `false` |
| `SavePdf` | هل تجمع الصور في PDF واحد؟ | `true` / `false` |
| `OutputFolder` | مجلد الحفظ | `C:\PrintOutput` |
| `Gamma` | تصحيح إضاءة الصورة (1.0 = طبيعي) | `0.8` إلى `1.3` |
| `Contrast` | تباين الصورة (1.0 = طبيعي) | `0.9` إلى `1.2` |
| `Brightness` | سطوع إضافي | `-50` إلى `+50` |

**كيف أعرف اسم الطابعة في Windows؟**

```
ابدأ → الإعدادات → الطابعات والماسحات → انقر على الطابعة
الاسم الظاهر في الأعلى هو ما تكتبه في WindowsPrinterName
```

### روابط API المتاحة

| الرابط | الوصف | الاستخدام |
|--------|-------|----------|
| `/api/health` | حالة الخادم | مراقبة تشغيل الخدمة |
| `/api/stats` | إحصائيات الجلسة | JSON |
| `/api/jobs?count=50` | آخر N مهمة | JSON |

| `/api/db/daily?days=30` | إحصائيات 30 يوم | JSON |
| `/api/db/jobs/csv` | سجل SQLite كـ CSV | تحميل ملف |

### كيف تستخدمها عملياً؟

**مراقبة يومية:**
→ افتح `http://الجهاز:9000` — ستعرف فوراً هل الخادم يعمل وكم طباعة تمت.

**تصدير التقارير الشهرية:**
→ انقر "📅 DB Daily" أو حمّل "📥 DB CSV" وافتحه في Excel.

**استكشاف الأعطال:**
→ انظر في جدول "آخر المهام" — العمليات الفاشلة تظهر باللون الأحمر مع رسالة الخطأ.

**قائمة الطابعات المثبتة:**
→ `http://الجهاز:9000/api/printers` تُظهر كل الطابعات المتاحة بحالتها.

## 7. أداة إدارة التراخيص (Admin GUI)

**الملف:** `DicomPrintAdminGui.exe` — هذا للمطور/الموزّع فقط، لا للمستخدم النهائي.

### الغرض

تُنشئ بها مفاتيح الترخيص لعملائك وتُديرها.

### أول تشغيل

```
1. شغّل DicomPrintAdminGui.exe
2. سيطلب منك ضبط "مفتاح رئيسي" (Master Key)
   → اختر كلمة مرور قوية واحتفظ بها
   → هذا المفتاح يحمي وصولك للأداة
3. ستفتح لوحة التحكم
```

### الشاشات والوظائف

#### لوحة التحكم (Dashboard)
يعرض:
- عدد المفاتيح الكلي / النشطة / المنتهية / الملغاة
- جدول آخر 20 مفتاح صادر

#### إنشاء مفتاح جديد (زر ➕)
```
الحقول:
  ✏️ اسم العميل/المؤسسة   → مثل: "مستشفى الملك فهد"
  📧 البريد الإلكتروني    → اختياري
  🏷️ نوع الترخيص         → Basic / Pro / Enterprise
  🔢 عدد العمليات         → 50 للتجربة، 500 للأساسي، غير محدود للمؤسسي
  📅 تاريخ الانتهاء       → لا ينتهي / أو تاريخ محدد
  ✅ الميزات المُفعَّلة   → PRINT / JPG / PDF / MULTI_PORT / WHATSAPP / REPORTS
  🔒 ربط بجهاز            → Hardware Lock (للحماية القصوى)
  💧 علامة مائية          → TRIAL للنسخة التجريبية

ثم:
  📄 الصق محتوى private_key.pem
  🔑 اضغط "إنشاء المفتاح"
  📋 انسخ المفتاح وأرسله للعميل
```

#### إدارة المفاتيح
- **عرض الجدول:** جميع المفاتيح مع حالتها (نشط/منتهي/ملغى)
- **إلغاء مفتاح:** إذا أردت منع عميل من الاستخدام
- **تمديد الصلاحية:** مد تاريخ انتهاء مفتاح موجود
- **تصدير CSV:** لملف Excel يحوي جميع العملاء

## 8. تحويل المشروع إلى EXE

### المتطلبات

- **Windows** مع **.NET 8 SDK** مثبّت
- Download من: https://dotnet.microsoft.com/download/dotnet/8.0

### بناء الخادم الرئيسي (`DicomPrintServer.exe`)

```cmd
cd src\DicomPrintServer

dotnet publish -c Release ^
  -r win-x64 ^
  --self-contained true ^
  -p:PublishSingleFile=true ^
  -p:IncludeNativeLibrariesForSelfExtract=true ^
  -o C:\Build\DicomPrintServer
```

**الناتج:** مجلد `C:\Build\DicomPrintServer` يحوي:
```
DicomPrintServer.exe      ← الملف الرئيسي (حوالي 60-80 MB)
appsettings.json          ← ملف الإعدادات (انسخه وعدّله)
```

### بناء أداة إدارة التراخيص (`DicomPrintAdminGui.exe`)

```cmd
cd src\DicomPrintAdminGui

dotnet publish -c Release ^
  -r win-x64 ^
  --self-contained true ^
  -p:PublishSingleFile=true ^
  -o C:\Build\DicomPrintAdminGui
```

**الناتج:** `DicomPrintAdminGui.exe` (هذا ملفك الشخصي، لا ترسله للعميل)

### بناء Admin CLI (أداة توليد مفاتيح RSA)

```cmd
cd src\DicomPrintAdminTool

dotnet publish -c Release ^
  -r win-x64 ^
  --self-contained true ^
  -o C:\Build\DicomPrintAdminTool
```

## 9. منظومة التوزيع

### الأطراف الثلاثة

```
┌──────────────────┐     يُصدر مفاتيح     ┌──────────────────┐     يوزّع برنامج     ┌──────────────────┐
│   المطور (أنت)   │ ──────────────────► │  الموزّع/Reseller │ ──────────────────► │  عميل نهائي      │
│                  │    (Admin GUI)        │                   │  (DicomPrintServer)  │  (مستشفى/عيادة)  │
└──────────────────┘                       └──────────────────┘                       └──────────────────┘
```

### ما يحتفظ به كل طرف

| الطرف | ما يملكه | ما لا يُشاركه |
|-------|---------|--------------|
| **المطور (أنت)** | `private_key.pem` + `DicomPrintAdminGui.exe` | المفتاح الخاص مطلقاً |
| **الموزّع** | `DicomPrintAdminGui.exe` | مفتاح رئيسي Admin GUI |
| **العميل** | `DicomPrintServer.exe` + `license.key` | — |

### خطوات الإعداد الأولي للمطور (مرة واحدة فقط)

```cmd
# 1. توليد زوج مفاتيح RSA
DicomPrintAdminTool.exe keygen

# الناتج:
#   private_key.pem  ← احتفظ به بأمان (لا ترسله لأحد أبداً)
#   public_key.pem   ← أضفه في كود LicenseManager.cs ثم أعد البناء
```

**خطوة حرجة:** افتح `LicenseManager.cs` وضع محتوى `public_key.pem` في:
```csharp
private const string PublicKeyPem = """
    -----BEGIN PUBLIC KEY-----
    MIIBIjAN... (مفتاحك الفعلي)
    -----END PUBLIC KEY-----
    """;
```
ثم أعد بناء `DicomPrintServer.exe`.

## 10. النسخة التجريبية

### ما تمنحه للعميل للتجربة

```
📦 مجلد التجربة:
   DicomPrintServer.exe    ← مبني بمفتاحك العام
   appsettings.json        ← معدّل بإعدادات مناسبة
   README_TRIAL.txt        ← تعليمات سريعة للعميل
```

**لا ترسل:**
- ❌ `license.key` → لأن غيابه يفعّل وضع Trial تلقائياً
- ❌ `private_key.pem`

### ماذا يحدث في وضع Trial (بدون license.key)

```
✅ البرنامج يعمل بالكامل
✅ كل المنافذ تعمل
✅ الطباعة تعمل
✅ JPG/PDF يعمل
✅ لوحة التحكم تعمل
⏱️ ينتهي بعد: 14 يوماً أو 50 عملية طباعة (أيهما أقرب)
⛔ عند الانتهاء: يتوقف عن الاستقبال تلقائياً
```

### ضبط مدة وحد التجربة

في `appsettings.json` الذي ترسله:
```json
"Trial": {
  "MaxOps": 50,
  "MaxDays": 14,
  "Enabled": true
}
```

| الحد | الوصف |
|------|-------|
| `MaxOps: 50` | 50 عملية طباعة |
| `MaxDays: 14` | 14 يوماً من أول تشغيل |
| `MaxDays: 30` | شهر كامل |
| `MaxOps: 200` | 200 عملية (أطول للعملاء الجادين) |

### ماذا يفعل العميل للتجربة؟

```
1. يستلم المجلد منك
2. يفتح appsettings.json ويضع اسم طابعته
3. يشغّل DicomPrintServer.exe
4. يضبط الجهاز الطبي (IP + AET + Port)
5. يجرّب الطباعة
6. عندما يعجبه البرنامج → يُرسل لك:
   ✉️ بصمة الجهاز (إذا طلبت Hardware Lock)
   ✉️ اسم المستشفى / المؤسسة
   ✉️ عدد المنافذ المطلوبة
   ✉️ الميزات المطلوبة (PDF / WhatsApp / إلخ)
```

### كيف يُرسل لك العميل بصمة الجهاز (Hardware ID)؟

```cmd
# العميل يشغّل هذا الأمر ويرسل لك الناتج:
wmic diskdrive get SerialNumber
# أو
wmic csproduct get UUID
```

## 11. ترقية للنسخة الكاملة

### ما تفعله بعد استلام طلب العميل

**في `DicomPrintAdminGui.exe`:**

```
1. افتح الأداة
2. اضغط ➕ "إنشاء مفتاح جديد"
3. أدخل:
   - اسم العميل: "مستشفى الملك فهد"
   - البريد: info@kfh.com
   - النوع: Pro (أو Enterprise)
   - عدد العمليات: 500 (أو غير محدود)
   - الانتهاء: بعد سنة (أو لا ينتهي)
   - الميزات: PRINT ✅ JPG ✅ PDF ✅ MULTI_PORT ✅
   - Hardware Lock: ✅ (الصق UUID الجهاز)
4. الصق محتوى private_key.pem
5. اضغط "إنشاء المفتاح"
6. انسخ المفتاح الناتج
```

**ثم أنشئ ملف `license.key`:**

```json
{
  "CustomerId":   "HOSP-KFH-001",
  "CustomerName": "مستشفى الملك فهد",
  "LicenseType":  "Full",
  "IssuedAt":     "2025-06-08T00:00:00Z",
```

## 12. الأسئلة الشائعة

---

**س: الخطأ "License allows max 1 port(s) but 2 configured" — ماذا يعني؟**

ج: كان هذا خطأً في نسخة قديمة — وضع Trial كان يُقيّد المنافذ. تم الإصلاح: الآن وضع Trial يسمح بجميع المنافذ المُهيَّأة. القيود في Trial هي فقط الوقت وعدد العمليات.

---

**س: ما الفرق بين Port 104 و 8000؟**

ج: المنفذ 104 هو المنفذ الرسمي لـ DICOM لكن يحتاج صلاحيات مسؤول نظام في Windows. المنافذ فوق 1024 (مثل 8000، 8001) تعمل بدون صلاحيات خاصة وهي الأفضل عملياً.

---

**س: الجهاز الطبي لا يتصل بالبرنامج.**

ج: تحقق من:
1. اسم AET في الجهاز الطبي **مطابق تماماً** لما في `appsettings.json`
2. IP الكمبيوتر صحيح في إعدادات الجهاز الطبي
3. المنفذ مفتوح في جدار الحماية (انظر القسم 4)
4. الخدمة تعمل: `sc query DicomPrintServer`

---

**س: لوحة التحكم لا تظهر.**

ج: تحقق من:
1. `AdminApi.Enabled: true` في `appsettings.json`
2. المنفذ 9000 مفتوح في الجدار الناري
3. اكتب `http://localhost:9000` وليس `https`

---

**س: الصور تطبع بلون داكن أو فاتح.**

ج: عدّل في `appsettings.json`:
```json
"Gamma": 1.2,       ← زد للصور الداكنة
"Brightness": 10,   ← زد للسطوع
"Contrast": 1.1     ← زد للتباين
```

---

**س: أريد طابعتين بإعدادات مختلفة.**

ج: أضف مدخلين في `Listeners` بأسماء AET وأرقام Port مختلفة. كل الأجهزة الطبية تُرسل للـ AET المحدد لها.

---

**س: كيف أعرف عدد الطباعات المتبقية في التجربة؟**

ج: في السجل عند التشغيل:
```
⚠️ TRIAL MODE - 13 day(s) / 37 operation(s) remaining
```
أو في لوحة التحكم: `/api/stats`

---

**س: هل يمكن نقل الترخيص لجهاز آخر؟**

ج: إذا لم تُفعّل Hardware Lock → نعم، انسخ `license.key` للجهاز الجديد.
إذا فعّلت Hardware Lock → لا، يجب إصدار ترخيص جديد بـ UUID الجهاز الجديد.

---

**س: أين سجل الطباعة؟**

ج:
```
قاعدة البيانات:  C:\ProgramData\DicomPrintServer\printlog.db
الصور المحفوظة: المجلد المحدد في OutputFolder
السجلات:        Windows Event Viewer → Application → DicomPrintServer
```

---

**س: هل البرنامج يعمل بدون إنترنت؟**

ج: نعم بالكامل. الإنترنت مطلوب فقط لـ:
- إشعارات WhatsApp (CallMeBot API)
- NTP لفحص صحة ساعة الجهاز في وضع Trial

---

**س: ماذا يحدث عند انقطاع الكهرباء؟**

ج: الخدمة مضبوطة على `start=auto` — تُعيد التشغيل تلقائياً عند عودة الكهرباء.
مهام الطباعة الجارية وقت الانقطاع تُفقد (الجهاز الطبي يُعيد الإرسال عادةً).
